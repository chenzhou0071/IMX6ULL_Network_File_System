**日期**: 2026-04-28

---

## 一、项目范围确认

### 1.1 要做什么

在正点原子 IMX6ULL 开发板上运行一个 C 语言服务器程序，Windows PC 上运行一个 QT C++ 客户端程序。两者通过自定义 TCP 二进制协议通信，实现：

- 文件浏览、上传、下载、删除
- 系统信息监控（CPU、内存、存储）
- 进程列表查看
- 用户登录认证

### 1.2 整体技术栈

| 组件 | 技术 |
|------|------|
| 服务器 | C语言 + POSIX线程 |
| 客户端 | C++ / QT5 |
| 通信协议 | 自定义TCP二进制协议 |
| 服务器编译 | arm-linux-gnueabihf-gcc（Windows交叉编译）|
| 客户端编译 | MinGW / MSVC（Windows本地编译）|

### 1.3 不做什么

- 不做 Samba 兼容（这是纯自研协议，不是蹭 SMB）
- 不做 Web 界面，纯桌面客户端
- 服务器暂时不做 systemd 守护进程（手动启动）
- 不做 OTA 升级、远程执行等高级功能

## 二、开发模式确认

### 2.1 开发流程

```
┌─────────────────────────────────────────────────────────┐
│  阶段一：服务器基础框架（先写）                          │
│  → telnet 验证端口能连                                  │
│  → Python 测试客户端验证登录、文件列表等各功能            │
│  → 服务器核心功能全部完成并验证通过                       │
│                                                         │
│  阶段二：QT客户端开发（后写）                            │
│  → 连接阶段一验证好的服务器                              │
│  → 开发各UI窗口完成完整功能                              │
└─────────────────────────────────────────────────────────┘
```

### 2.2 测试方式

- **板子**：独立运行服务器程序，和 Windows PC 在同一局域网
- **调试工具**：Python 写一个轻量测试客户端（几百行），在 Windows 上跑，专门用来发协议包测试服务器
- **拷贝方式**：交叉编译后手动拷贝到 U盘 → 插板子运行

### 2.3 开发顺序

**阶段一：服务器（优先完成）**

1. 项目目录结构 + Makefile（交叉编译配置）
2. 网络协议头文件（统一协议定义）
3. 服务器框架：Socket封装、线程池、多客户端管理
4. 用户认证（配置文件解析 + 登录）
5. 文件操作：目录列表
6. 文件操作：上传、下载（分块传输）
7. 文件操作：删除、路径安全检查
8. 系统信息采集（CPU、内存、磁盘）
9. 进程列表采集
10. 心跳机制、超时管理
11. 日志模块

**阶段二：Python 测试客户端**

- 实现所有协议命令的发送和接收验证

**阶段三：QT 客户端**

1. 协议编解码模块
2. 网络通信模块
3. 主窗口（首页 + 连接管理）
4. 文件管理器窗口
5. 传输进度对话框
6. 系统监控窗口
7. 日志查看器窗口
8. 系统设置对话框

## 四、协议关键设计确认

- **魔数**: `0xA5A5A5A5`
- **帧头**: 固定 16 字节（magic 4 + length 4 + cmd 2 + seq 2 + checksum 4）
- **字节序**: 小端序
- **最大帧长**: 64KB
- **心跳间隔**: 30秒一次
